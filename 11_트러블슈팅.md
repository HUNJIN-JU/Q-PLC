## 1. LED 에러 진단

### CPU LED 확인

| LED | 상태 | 원인 | 대책 |
|-----|------|------|------|
| POWER OFF | 전원 이상 | 전원 유니트 확인, 전압 측정 |
| RUN OFF | CPU 정지 | RUN/STOP 스위치 확인 |
| ERR. 점등 | 중대 에러 | SD0 에러 코드 확인 후 대응 |
| ERR. 점멸 | 경미 에러 | 프로그램/파라미터 확인 |
| BAT. 점등 | 배터리 저하 | Q6BAT 교체 |
| USER 점등 | 사용자 에러 | 프로그램 내 에러 체크 |

## 2. 주요 에러 코드

### CPU 에러

| 코드 | 명칭 | 원인 | 대책 |
|------|------|------|------|
| 1000 | WDT 에러 | 스캔 타임 초과 | WDT 설정값 증가 또는 프로그램 최적화 |
| 2000 | 메모리 에러 | ROM/RAM 이상 | CPU 교체 |
| 3000 | I/O 에러 | I/O 모듈 이상 | 모듈 장착 확인, 교체 |
| 4000 | SP. 유니트 에러 | 특수 모듈 이상 | 모듈 파라미터 확인 |
| 5000 | 파라미터 에러 | 설정 오류 | GX Works2에서 재설정 |
| 6000 | 프로그램 에러 | 프로그램 오류 | 에러 위치 확인 후 수정 |

### 프로그램 에러 상세

| 코드 | 명칭 | 원인 |
|------|------|------|
| 6100 | 연산 에러 | 0으로 나누기 등 |
| 6200 | FEND 없음 | 메인 프로그램 끝에 FEND 미작성 |
| 6210 | FOR-NEXT 불일치 | FOR/NEXT 쌍 안 맞음 |
| 6220 | MC-MCR 불일치 | MC/MCR 네스팅 오류 |

## 3. 에러 확인 방법

### GX Works2에서 확인
1. **진단 > PLC 진단** → 현재 에러 확인
2. **진단 > 에러 이력** → 과거 에러 이력 조회
3. 에러 코드 더블클릭 → 에러 발생 위치로 이동

### 특수 레지스터로 확인
```
SD0   : 현재 에러 코드
SD1   : 에러 발생 유니트 정보
SD16  : 에러 발생 프로그램 명
SD50  : 에러 이력 수
```

## 4. 일반적 트러블슈팅

### 증상별 체크리스트

#### PLC 전원이 들어오지 않음
- [ ] 입력 전원 전압 확인 (AC100~240V 또는 DC24V)
- [ ] 전원 유니트 커넥터 확인
- [ ] 퓨즈 확인
- [ ] 배선 극성 확인 (DC 타입)

#### 프로그램이 동작하지 않음
- [ ] RUN/STOP 스위치 → RUN 위치 확인
- [ ] RUN LED 점등 확인
- [ ] ERR. LED 확인
- [ ] 프로그램 전송 여부 확인
- [ ] 입력 신호 모니터링 → 실제 ON/OFF 확인

#### 특정 출력이 동작하지 않음
- [ ] 출력 LED 확인 (PLC 내부 ON인지)
- [ ] 외부 배선 확인 (단선, 접촉 불량)
- [ ] 퓨즈 확인 (릴레이 출력 유니트)
- [ ] 부하 확인 (소손, 단선)
- [ ] 래더 모니터로 출력 코일 ON 조건 확인

#### 통신이 되지 않음
- [ ] 케이블 연결 상태 확인
- [ ] IP 주소/국 번호 중복 확인
- [ ] 통신 파라미터 일치 확인 (속도, 프로토콜)
- [ ] LED 상태 확인 (RUN, ERR, SD/RD)

## 5. 예방 보전

### 정기 점검 항목

| 주기 | 점검 항목 |
|------|-----------|
| 매일 | LED 상태, 이상 발열/소음 확인 |
| 매월 | 배선 단자 조임 확인, 먼지 청소 |
| 6개월 | 배터리 전압 확인, 팬 필터 청소 |
| 1년 | 릴레이 출력 접점 수명 확인 |
| 3~5년 | 배터리 교체 (Q6BAT) |

### 프로그램 백업
- **SD 카드**: 파라미터 + 프로그램 백업
- **GX Works2**: PC에 프로젝트 저장
- **ROM 운전**: 표준 ROM에 백업 후 ROM 운전 설정
